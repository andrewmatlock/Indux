### Overview

Indux centralizes global theme management with a curated set of CSS variables, sometimes referred to as design tokens. These variables are referenced throughout Indux's base HTML styles, common utility classes, and can be compiled as custom utilities—establishing your project's visual identity with minimal markup.

<Note>Every Indux stylesheet except [indux.reset.css](/styles/reset) references the theme's CSS variables. If a CSS variable is removed, stylesheets will use a static fallback value.</Note>

---

## Setup

The theme is available in the full Indux CSS library, or as a standalone stylesheet for use with specific elements or utilities.

<CodeGroup>
```html Indux CSS
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.css">
```
```html Standalone
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.theme.css">

<!-- Example of standalone styles with theme references -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.buttons.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.utilities.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.typography.css">
```
</CodeGroup>

## Theme Variables

<Note>Certain variable names use namespace prefixes like `--color-` to automatically generate associated utility classes in projects utilizing [Tailwind v4+](https://tailwindcss.com/docs/theme#theme-variable-namespaces). Example utilities are provided below.</Note>

### Color Palette
In the default theme, all other color variables in both light and dark themes (except primaries, accents, and overlays) make reference to the color palette as a single source of truth. Consider grabbing [alternate palettes](https://tailwindcss.com/docs/colors) or [generating your own](https://uicolors.app/generate).

| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--color-50` to<br/> `--color-950` | Base color scale from lightest to darkest | `bg-color-500`<br/>`text-color-700`<br/>`border-color-300` |


### Theme Colors
See [color themes](/plugins/color-themes) for more information on setting up light/default and dark themes. Use the `.dark { ... }` selector to override light/default color values with their dark equivalent.

| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--color-page` | Page background color | `bg-page`<br/>`text-page` |
| `--color-surface-1` | Initial surface background | `bg-surface-1` |
| `--color-surface-2` | Next surface background | `bg-surface-2` |
| `--color-surface-3` | Last surface background | `bg-surface-3` |
| `--color-content-stark` | High contrast text color | `text-content-stark` |
| `--color-content-neutral` | Medium contrast text color | `text-neutral` |
| `--color-content-subtle` | Low contrast text color | `text-content-subtle` |
| `--color-field-surface` | Background color for interactive elements | `bg-field-surface` |
| `--color-field-surface-hover` | Hover state background for interactive elements | `hover:bg-field-surface-hover` |
| `--color-field-inverse` | Content color for interactive elements | `text-field-inverse` |
| `--color-popover-surface` | Modal and dropdown background | `bg-popover-surface` |
| `--color-line` | Border and divider color | `border-line` |

### Semantic Colors
These semantic colors are easily applied to text, backgrounds, and certain form elements using the Indux [utility](/styles/utilities) classes `primary`, `accent`, and `danger`.

| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--color-primary` | Primary brand color | `bg-primary`<br/>`border-primary` |
| `--color-primary-inverse` | Content color on primary backgrounds | `text-primary-inverse` |
| `--color-primary-text` | Primary text color | `text-primary-text` |
| `--color-accent` | Accent color for highlights | `bg-accent`<br/>`border-accent` |
| `--color-accent-inverse` | Content color on accent backgrounds | `text-accent-inverse` |
| `--color-accent-text` | Accent text color | `text-accent-text` |
| `--color-danger` | Error and danger states | `bg-danger`<br/>`border-danger` |
| `--color-danger-inverse` | Content color on danger backgrounds | `text-danger-inverse` |
| `--color-danger-text` | Danger text color | `text-danger-text` |

### Overlay Colors
| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--color-overlay-dark` | Dark overlay for banners | `bg-overlay-dark` |
| `--color-overlay-light` | Light overlay for banners | `bg-overlay-light` |

### Spacing & Sizing
| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--radius` | Default border radius | — |
| `--spacing` | Base spacing unit | `p-1` `m-1` `gap-1` |
| `--spacing-field-padding` | Form field padding | `p-field-padding` |
| `--spacing-field-height` | Form field height | `h-field-height` |
| `--spacing-popover-offset` | Dropdown & tooltip positioning offset | `mt-dropdown-offset` |

### Effects
| Variable | Purpose | Ex. Tailwind Utilities |
|----------|---------|-------------------|
| `--transition` | Default form element transition timing | — |

```css
    --color-700: oklch(28.7% 0.030787 270.1);
    --color-800: oklch(20.7% 0.026326 268.7);
    --color-900: oklch(16.6% 0.026 267);
    --color-925: oklch(14.49% 0.0115 266.85);
    --color-950: oklch(3.89% 0.0181 262.25);

    /* Light theme */
    --color-page: var(--color-50);
    --color-surface-1: var(--color-100);
    --color-surface-2: var(--color-200);
    --color-surface-3: var(--color-300);

    --color-content-neutral: var(--color-600);

    --color-field-surface: color-mix(in oklch, var(--color-content-stark) 7%, transparent);
    --color-field-surface-hover: color-mix(in oklch, var(--color-content-stark) 11%, transparent);
    --color-field-inverse: var(--color-content-stark);
    --color-popover-surface: var(--color-page);
    --color-line: color-mix(in oklch, var(--color-content-stark) 11%, transparent);
    --color-primary: var(--color-content-stark);
    --color-primary-inverse: var(--color-page);
    --color-primary-text: var(--color-content-stark);
    --color-accent: oklch(90.56% 0.1724 148.7);
    --color-accent-inverse: var(--color-950);
    --color-accent-text: oklch(70.7% 0.229867 143.9);
    --color-danger: oklch(63.68% 0.2078 25.33);
    --color-danger-inverse: oklch(100% 0 0);
    --color-danger-text: var(--color-danger);

    /* Overlays */
    --color-overlay-dark: oklch(50% 0 0);
    --color-overlay-light: oklch(50% 0 100);

    /* Sizes */

    --spacing: 0.25rem;
    --spacing-field-padding: calc(var(--spacing) * 2.5);
    --spacing-field-height: calc(var(--spacing) * 9);
    --spacing-popover-offset: calc(var(--spacing) * 2);

    /* Effects */
    --transition: all 0.1s ease-in-out;
}

/* Dark theme overrides */
.dark {
    --color-page: var(--color-950);
    --color-surface-1: var(--color-900);
    --color-surface-2: var(--color-800);
    --color-surface-3: var(--color-700);
    --color-popover-surface: var(--color-700);
    --color-content-stark: var(--color-50);
    --color-content-neutral: var(--color-400);
    --color-content-subtle: var(--color-500);
    --color-accent-text: oklch(90.56% 0.1724 148.7);
}

@layer base {

    /* Default font and colors */
    :where(html),
    :host {
        line-height: 1.5;
        font-family: ui-sans-serif, system-ui, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji';
        color: var(--color-content-stark, inherit);
        background-color: var(--color-page, inherit)
    }

    /* Text selection */
    ::selection {
        background-color: color-mix(in oklch, var(--color-surface-1) 92%, var(--color-content-stark))
    }

    /* Focus state */
    :where(:focus-visible) {
        outline: 2px solid color-mix(in oklch, var(--color-field-surface) 90%, var(--color-field-inverse));
        outline-offset: 0
    }
}
```

## Custom Utilities

Theme variables can be compiled into Tailwind-style custom utility classes using Indux's utilities script. This can be added to your project with all [Indux plugins](/getting-started/setup) or as a standalone script.

```html
<script src="https://cdn.jsdelivr.net/npm/indux/dist/indux.utilities.js"></script>
```

Unlike other plugins, this script is not associated with Alpine and does not require Alpine to operate.


### Application
Any variables in any `:root` or `@theme` declarations throughout your project will be compiled into utility classes, provided they use Tailwind [namespace](https://tailwindcss.com/docs/theme#theme-variable-namespaces) prefixes like `--color-` or `--spacing-`. For example:

```css
:root {
    --color-primary: red;
}
```

...can compile into classes like:

```css
.bg-primary { background-color: red; }
.text-primary { color: red; }
.border-primary { border-color: red; }
```

The runtime compiler is highly optimized for performance. Running concurrent to other page load events, it only generates CSS for classes used in the current DOM view, and leverages caching to avoid redundant work. As a result, updating of the generated `<style>` tag in the head is nearly instantaneous, with average execution times under 10ms.

### Tailwind Integration

The utility plugin operates independently, but is designed to pair well with [Play CDN](https://tailwindcss.com/docs/installation/play-cdn), a client-side version of Tailwind. While it's not broadcast for production use, Play CDN aligns with Indux's ethos of being plug-and-play with no build steps, and has a typically nelgible impact on performance. Add Play CDN to your project using Indux's all inclusive script (bundled with all Indux plugins alongside Alpine), or via Tailwind's CDN link.

<CodeGroup>
```html All Inclusive
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.alpine.tailwind.css">
```
```html Tailwind CDN Link
<head>
    ...
    <script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>
    <script src="https://cdn.jsdelivr.net/npm/indux/dist/indux.utilities.js"></script>
    ...
</head>
```
</CodeGroup>

The all inclusive script's embedded Play CDN has been modified to remove Tailwind's Preflight styles in favour of Indux's [reset](/styles/reset) styles. It also removes a console warning about using Play CDN in production.

<Note>Projects with Tailwind v4+ [directly installed](https://tailwindcss.com/docs/installation/using-vite) will already compile theme variables from [@theme](https://tailwindcss.com/docs/theme) declarations during the build step. In this scenario, Indux's all inclusive script or the standalone `indux.utilities.js` are redundant and should not be used. Be sure to update your theme's `:root { ... }` declarations to `@theme { ... }` for compatibility.</Note>
