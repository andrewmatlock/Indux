### Overview

Indux includes a native routing system that uses HTML files for pages and `manifest.json` to define their routes, attributes, and other properties. If no pages are defined, `index.html` is rendered by default.

---

### Create Pages

HTML page files can be placed anywhere in the project directory. Their contents can follow the standard HTML structure seen in `index.html`, however Indux only requires the `<body>` tag:

```html home.html
<body>
    <h1>Hello world</h1>
</body>
```

If a head is included, its content will be appended to the global head when the page is loaded, and removed when unloaded:

```html home.html
<head>
    <title>Home</title>
    <link rel="stylesheet" href="home.css">
    <script src="home.js"></script>
</head>

<body>
    <h1>Hello world</h1>
</body>
```

<Note>Page-specific head content can also be declared in the manifest.</Note>

---

### Register Pages

Pages must be registered in `manifest.json` under the `pages` array:

```json manifest.json
"pages": [
    {
        "source": "html/home.html",
        "path": "/"
    }
],
```

The `source` property is the path to the page file relative to the project root. The `path` property is the URL path where the page will be accessible.

<Warning>`manifest.json` will break routing and rendering with a single syntax error. Be sure to validate your JSON before saving.</Warning>

---

#### Optional Properties

Other properties can be optionally applied to further configure the page:

| Property | Type | Description |
|----------|------|-------------|
| `title` | string | Page title used for meta tags and browser title. |
| `description` | string | Page description used for meta tags. |
| `author` | string | Page author used for meta tags. |
| `fallback` | boolean | If true, the page will be used as the 404 fallback page. The "/" path is otherwise used as the default fallback. |
| `head` | object | Custom head content specific to this page. Can include `meta` tags, `scripts`, and `links` that will be injected into the document head. |
| `pathContent` | string \| string [] | Content type(s) to load based on URL path segments. See [Template Pages](/abilities/template-pages). |

For example:

```json manifest.json
"pages": [
    {
        "source": "html/home.html",
        "path": "/",
        "title": "Home",
        "description": "There's no place like home.",
        "author": "Agoraphobic"
    },
    {
        "source": "html/about.html",
        "path": "/about",
        "collections": [
            "team"
        ]
    }
],
```

### Global Head

Global head content will apply to all pages. It can be defined directly in the `index.html` head, or in `manifest.json` under the `defaultMeta` property:

```json manifest.json
"defaultMeta": {
    "title": "My Project",
    "author": "Me",
    "description": "Lorem ipsum dolor sit amet.",
    "image": "/images/default-og-image.jpg",
    "ogType": "website"
},
```

| Property | Type | Description |
|----------|------|-------------|
| `title` | string | Default page title used for meta tags and browser title. |
| `author` | string | Site author used for meta tags. |
| `description` | string | Default meta description used for SEO and social sharing. |
| `image` | string | Default social share image URL. |
| `ogType` | string | Open Graph type, defaults to "website". |

<Note>Page-specific properties take precedence over global properties.</Note>
