## Overview

Dropdowns in Indux are `<menu>` elements that can be triggered by clicks or hover interactions. Built on the HTML Popover API, they provide accessible navigation menus, context menus, and nested dropdowns with automatic positioning and keyboard navigation.

## Setup

Add all [Indux plugins and CSS](/getting-started/setup) to your project, or use the standalone dropdown plugin and CSS files:

```html
<!-- Alpine -->
<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>

<!-- Indux Dropdowns -->
<script src="https://cdn.jsdelivr.net/npm/indux/dist/indux.dropdowns.js"></script>
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/indux/dist/indux.dropdowns.css">
```

## Basic Dropdown

Create a simple dropdown with the `x-dropdown` directive:

<Frame>
<button>Open Menu</button>
<menu>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</menu>
</Frame>

```html
<button x-dropdown="basic-menu">Open Menu</button>

<menu popover id="basic-menu">
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</menu>
```

The dropdown automatically positions itself relative to the trigger element and handles keyboard navigation.

---

## Hover Dropdowns

Add the `hover` modifier for mouse-triggered dropdowns:

<Frame>
<button>Hover Me</button>
<menu>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</menu>
</Frame>

```html
<button x-dropdown.hover="hover-menu">Hover Me</button>

<menu popover id="hover-menu">
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</menu>
```

Hover dropdowns include a delay to prevent accidental opening and auto-close when the mouse leaves the trigger and menu area.

## Positioning

Control dropdown positioning with direction and alignment modifiers:

### Direction Modifiers

<CodeGroup>
```html
<button x-dropdown.top="menu">Top</button>
<button x-dropdown.bottom="menu">Bottom</button>
<button x-dropdown.left="menu">Left</button>
<button x-dropdown.right="menu">Right</button>
```
```html
<button x-dropdown.top.right="menu">Top Right</button>
<button x-dropdown.bottom.start="menu">Bottom Start</button>
<button x-dropdown.left.end="menu">Left End</button>
```
</CodeGroup>

### Alignment Options

- **`start`** - Aligns to the start of the trigger element
- **`end`** - Aligns to the end of the trigger element
- **`left`** - Aligns to the left side (for top/bottom dropdowns)
- **`right`** - Aligns to the right side (for top/bottom dropdowns)

### Default Positioning

- **Top-level dropdowns**: Positions to the bottom start of the trigger.
- **Nested dropdowns**: Position to the right start of the trigger.

## Nested Dropdowns

Create multi-level navigation menus with nested dropdowns:

<Frame>
<button>Main Menu</button>
<menu>
    <li>Main Item 1</li>
    <button>Submenu 1</button>
    <li>Main Item 3</li>
    <menu>
        <li>Sub Item 1</li>
        <button>Submenu 2</button>
        <li>Sub Item 3</li>
        <menu>
            <li>Deep Item 1</li>
            <li>Deep Item 2</li>
            <li>Deep Item 3</li>
        </menu>
    </menu>
</menu>
</Frame>

```html
<button x-dropdown="nested-menu">Main Menu</button>

<menu popover id="nested-menu">
    <li>Main Item 1</li>
    <button x-dropdown.hover="submenu-1">Submenu 1</button>
    <li>Main Item 3</li>
    <menu popover id="submenu-1">
        <li>Sub Item 1</li>
        <button x-dropdown.hover="submenu-2">Submenu 2</button>
        <li>Sub Item 3</li>
        <menu popover id="submenu-2">
            <li>Deep Item 1</li>
            <li>Deep Item 2</li>
            <li>Deep Item 3</li>
        </menu>
    </menu>
</menu>
```

Nested dropdowns automatically position themselves to avoid overlapping and maintain proper navigation flow.

## Keyboard Navigation

Dropdowns support full keyboard navigation:

- **Tab/Shift+Tab** - Navigate between focusable elements
- **Escape** - Close the dropdown and return focus to trigger
- **Arrow keys** - Navigate menu items (when implemented)

The dropdown automatically manages focus and closes when navigation moves outside the menu area.

---

## Accessibility

Dropdowns are built with accessibility in mind:

- **ARIA attributes** - Automatically applied for screen readers
- **Focus management** - Proper focus trapping and restoration
- **Keyboard support** - Full keyboard navigation
- **Popover API** - Native browser accessibility features

---

## Styling

Dropdowns use CSS custom properties for consistent spacing:

```css
:root {
    --spacing-popover-offset: 0.5rem;
}
```

Customize the dropdown offset by modifying this variable in your CSS.

<Warning>Avoid mixing `x-dropdown.hover` and `x-tooltip.hover` in the same trigger element. Dropdowns and tooltips both utilize the HTML `popover` attribute, where only one can display at a time.</Warning>
